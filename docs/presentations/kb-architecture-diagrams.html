Strengths

  • Always provides context — no missed queries
  • Zero user effort — fully automatic
  • Simple to implement and maintain

Limitations

  • No filtering by date, source, or topic
  • Can't browse, paginate, or discover
  • May inject irrelevant context (wastes tokens)

Strengths

  • Structured queries — filter by date, source, topic
  • Only queries KB when needed (saves tokens)
  • Multi-step reasoning and chained tool calls

Limitations

  • LLM may skip the tool call → no KB context
  • Extra round-trip latency (LLM → tool → LLM)
  • Requires careful prompt engineering
Our Approach

Hybrid — Best of Both Worlds

Automatic RAG baseline + on-demand tool calls for deep research

Only if needed
→
✅
Rich answer
Full coverage

Why Hybrid Wins

  • No missed queries — RAG always provides a baseline
  • Deep research — tools add filters when needed
  • Graceful degradation — if tools fail, RAG still works
  • Best accuracy — Semantic + BM25 + structured queries

Decision Logic

  • RAG only: "What is OAuth?" → chunks are enough
  • RAG + Tool: "Auth docs from 2025" → date filter
  • RAG + Tool: "List all topics" → browse tool
  • RAG + Tool: "Compare OAuth vs API keys" → research
