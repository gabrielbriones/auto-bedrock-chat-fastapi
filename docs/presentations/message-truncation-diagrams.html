Performance Metrics (for a 600K document example message)

⚡ Processing Speed
Plain text is instant string manipulation. AI requires 3 sequential LLM API calls with network latency and processing time per call.
Plain Text Truncation < 1ms
String operations only — no API calls, instant result
AI Summarization ~30 sec
3 sequential LLM calls with network latency (~10 seconds each)
💰 Token Cost
Plain text is free. AI requires 3 LLM calls (200K chars/~50K tokens each) = ~150K input tokens + ~37K output tokens. Using Claude 3.5 Sonnet pricing: ($3 input + $15 output per 1M tokens) = ~$1.00 total cost.
Plain Text Truncation $0.00
No API calls, no token consumption — completely free
AI Summarization ~$1.00
3 LLM calls: ~150K input + ~37K output tokens = $0.45 input + $0.55 output
🎯 Information Retention
Plain text keeps 70% of characters (head+tail) but loses critical middle content. AI keeps only 10% of characters (60K) but preserves ~92% (estimated) of key information through intelligent summarization.
The "lost" 8%: Redundant explanations, verbose descriptions, conversational filler, and formatting artifacts — intentionally filtered out, not critical data loss.
Plain Text Truncation 65%
425K chars kept (340K head + 85K tail), but middle 175K with potential critical data is lost
AI Summarization ~92% (est.)
Estimated retention based on typical LLM summarization. Preserves: facts, numbers, names, decisions. Filters: redundancy, verbosity.

Rolling Accumulation Strategy
Each iteration combines the previous summary with the next chunk, creating a rolling summary that preserves context across the entire document. The final iteration uses the target size constraint to ensure the output fits within the budget.
Example: 600K chars split into 3 chunks (200K each) → 3 LLM calls → 60K char final summary (10× reduction)
